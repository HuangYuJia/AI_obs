### 二、Lucy API 集成

#### 2.1 获取 Lucy API 信息
- 从 X (Twitter) 找到日本开发者的实时换装系统截图
- 发现使用的是 `lucy-vton-3` 模型
- 通过搜索确认是 Decart 公司的 Lucy 2.1 API

#### 2.2 Lucy API 文档研究
- 官网：https://decart.ai
- API 平台：https://platform.decart.ai
- 文档：https://docs.platform.decart.ai

**Lucy 2.1 两种模式：**

| 模式 | 接口 | 速度 | 适用场景 |
|------|------|------|---------|
| 批量处理 | REST API | 30-60秒/帧 | 视频编辑 |
| 实时模式 | WebRTC | 实时 | 直播 |

#### 2.3 Lucy API 集成过程

**步骤 1：安装 SDK**
```bash
pip install decart
```

**步骤 2：创建 lucy_api.py**
- 使用 REST API (`/v1/jobs/lucy-2.1`)
- 支持图片转视频（API 需要视频输入）
- 异步任务轮询机制
- 结果缓存

**步骤 3：认证方式**
- 使用 `x-api-key` 请求头
- API Key 格式：`dct_xxxxx`

**步骤 4：API 工作流程**
```
1. POST /v1/jobs/lucy-2.1 → 提交任务
2. GET /v1/jobs/{job_id} → 轮询状态
3. GET /v1/jobs/{job_id}/content → 下载结果
```

#### 2.4 测试结果
- API 连接成功 ✓
- 图片上传成功 ✓
- 任务提交成功 ✓
- 结果下载成功 ✓
- 单张处理时间：约 50-60 秒

### 三、OBS 视频流优化

#### 3.1 问题描述
- OBS 视频画面卡顿
- AI 处理时画面冻结

#### 3.2 优化方案

**截图优化：**
| 参数 | 之前 | 优化后 |
|------|------|--------|
| 分辨率 | 1920x1080 | 640x360 |
| 格式 | PNG | JPEG |
| 质量 | 80% | 50% |

**前端优化：**
- 使用 `requestAnimationFrame` 替代 `setInterval`
- 智能节流：15 FPS 上限
- AI 处理时降帧：~3 FPS
- 防止重复请求

**后端优化：**
- 线程池处理 AI 请求
- 不阻塞截图服务
