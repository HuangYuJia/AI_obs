## 1. Lucy VTON 实时模式集成

### 1.1 调研 Decart SDK
- 发现 Decart 提供 Python SDK (`decart[realtime]`)，支持 WebRTC 实时模式
- 实时模式模型：`models.realtime("lucy-vton-latest")`，分辨率 1088x624，30fps
- 不需要特殊权限申请，现有 API Key 即可使用
- 定价：2 credits/秒（720p）

### 1.2 安装依赖
```bash
pip install decart[realtime]
```
自动安装了 `livekit`、`tenacity`、`protobuf` 等依赖。

### 1.3 重写 `lucy_realtime.py`
- 原实现使用 `process()` 做单帧处理（错误方式）
- 新实现使用 `RealtimeClient.connect()` 建立 WebRTC 持久连接
- 核心方法：
  - `connect()` — 建立 WebRTC 会话，设置初始服装参考图
  - `push_frame()` — 推送 OBS 帧到 VideoSource
  - `update_clothing()` — 中途切换服装（原子操作 `SetInput`）
  - `_consume_remote_track()` — 消费处理后的帧
  - `_frame_to_pil()` — LiveKit VideoFrame 转 PIL Image

### 1.4 添加 WebSocket 端点 `/ws/vton`
- 在 `server.py` 中新增 WebSocket 端点
- 支持消息类型：`start`、`frame`、`update`、`stop`
- 帧通过 base64 编码传输

### 1.5 前端 WebSocket 集成
- 新增 `connectVTONWebSocket()`、`disconnectVTONWebSocket()`、`handleVTONMessage()`
- 修改 `applyRealTimeTryOn()` 使用 WebSocket 替代 REST 轮询
- `startGeneration()` 时建立连接，`stopGeneration()` 时断开

## 2. Bug 修复

### 2.1 提示词问题
- **问题**：空提示词导致 `InvalidInputError: Prompt cannot be empty`
- **解决**：默认提示词改为 "try on"
- **发现**：描述人物外观的提示词会让 AI 生成新图而非换装，VTON 提示词应描述换装动作

### 2.2 API Key 加载问题
- **问题**：系统环境变量中的旧 Key 覆盖了 .env 文件的新 Key
- **原因**：`load_dotenv()` 默认不覆盖已存在的环境变量
- **解决**：改为 `load_dotenv(override=True)`

### 2.3 帧处理时序问题
- **问题**：`on_remote_stream` 回调在 `is_connected=True` 之前触发
- **解决**：`_consume_remote_track` 中增加等待连接建立的逻辑

### 2.4 重复 WebSocket 连接
- **问题**：前端重复创建 WebSocket 连接导致会话断开
- **解决**：增加连接状态检查，防止重复连接

### 2.5 btnVolume 元素不存在
- **问题**：删除底部控件后，JS 仍引用 `btnVolume` 导致初始化报错
- **解决**：移除相关引用和事件监听
